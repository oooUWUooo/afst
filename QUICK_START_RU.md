# Быстрый старт (Quick Start) - Русский

## Проблема: "Login failed: NetworkError when attempting to fetch resource"

Если вы видите эту ошибку, это означает, что приложение не может подключиться к backend-серверу. GitHub Pages - это статический хостинг, который не может обрабатывать API-запросы. Ниже приведены пошаговые инструкции для решения проблемы.

## Вариант 1: Быстрый запуск с Render.com (рекомендуется)

### Шаг 1: Деплой backend на Render.com

1. **Зарегистрируйтесь на [Render.com](https://render.com)**
2. **Создайте новый Web Service**:
   - Нажмите "New +" → "Web Service"
   - Выберите ваш GitHub репозиторий с этим проектом
   - Установите:
     - Environment: `Docker`
     - Branch: `main`
     - Region: `Frankfurt` (или ближайший к вам)
     - Instance Type: `Free`
3. **Нажмите "Create Web Service"**
4. **Дождитесь завершения деплоя** (может занять 5-10 минут)
5. **Скопируйте URL вашего backend** (например, `https://your-project.onrender.com`)

### Шаг 2: Настройка frontend на GitHub Pages

1. **Запуште этот проект в ваш GitHub репозиторий**
2. **Включите GitHub Pages**:
   - Перейдите в Settings → Pages
   - Выберите "Deploy from a branch"
   - Выберите ветку `main`
   - Нажмите "Save"
3. **Скопируйте URL вашего GitHub Pages** (например, `https://yourusername.github.io/your-repo-name`)

### Шаг 3: Настройка подключения

1. **Откройте ваш GitHub Pages сайт в браузере**
2. **Найдите раздел "API Configuration"**
3. **Введите URL вашего backend с Render.com** (без слэша в конце)
4. **Нажмите "Save URL"**
5. **Теперь вы можете регистрироваться и входить в систему**

## Вариант 2: Локальный запуск для тестирования

Если вы хотите протестировать локально:

1. **Установите зависимости**:
   ```bash
   pip install fastapi uvicorn python-multipart
   ```

2. **Запустите backend**:
   ```bash
   uvicorn example_backend:app --reload
   ```

3. **Запустите frontend в другом терминале**:
   ```bash
   python start_server.py
   ```

4. **Откройте `http://localhost:8080` в браузере**
5. **В разделе "API Configuration" введите `http://localhost:8000`**
6. **Нажмите "Save URL" и начните использовать приложение**

## Проверка работоспособности

После настройки:

1. **Зарегистрируйтесь** с помощью формы "Register"
2. **Войдите в систему** с помощью формы "Login"
3. **Попробуйте добавить книгу** через форму "Add New Book"
4. **Попробуйте добавить читателя** через форму "Add New Reader"
5. **Попробуйте выдать книгу** через форму "Borrow a Book"

Если все работает, значит вы успешно настроили систему!

## Устранение неполадок

Если все еще возникают проблемы:

1. **Проверьте URL backend** - убедитесь, что он введен правильно
2. **Проверьте работоспособность backend** - откройте `https://your-project.onrender.com/docs`
3. **Очистите localStorage** браузера и введите URL снова
4. **Убедитесь, что CORS настроен** в backend (в примере уже настроен)

## Важно

- Данные в примере backend хранятся в памяти и будут сброшены при перезапуске сервера
- Для продакшена рекомендуется использовать базу данных
- Не забывайте про безопасность паролей в продакшене
